\# Data Directory – Traceability \& Ownership



This directory contains all input and generated data used by the

CABLEGNOSIS Dash application.



No application logic should assume hard-coded paths.

All paths are defined centrally in `utils/paths.py`.



---



\## raw/



\*\*Purpose\*\*

\- Manually provided datasets

\- External imports

\- Static reference data



\*\*Notes\*\*

\- Not modified by the application

\- Safe to version-control if needed



---



\## generated/



All files in this directory are \*\*generated by the application\*\*.

They may be deleted or regenerated at any time.



\### generated/synthetic/



\*\*Produced by\*\*

\- `logic/synthetic\_generator.py`



\*\*Triggered from\*\*

\- `tabs/generator\_tab.py`

\- `tabs\_core/interactive\_timeline.py`



\*\*Purpose\*\*

\- Synthetic time-series data for testing and demos



---



\### generated/timeline/



\*\*Produced by\*\*

\- `tabs\_core/interactive\_timeline.py`



\*\*Files\*\*

\- `interactive\_tab\_dataset.json`



\*\*Purpose\*\*

\- Time-based visualization for the Data Timeline Viewer tab



---



\### generated/monitoring/



\*\*Produced by\*\*

\- `tabs/monitoring\_tab.py`



\*\*Files\*\*

\- `ucy\_load.csv`

\- `ucy\_temp.csv`



\*\*Purpose\*\*

\- Raw SCADA-like buffers for real-time monitoring demos



\*\*Notes\*\*

\- CSVs are raw input buffers

\- Normalization and aggregation are applied on READ, not on write



---



\### generated/uptime/



\*\*Produced by\*\*

\- Monitoring / KPI demo logic



\*\*Purpose\*\*

\- Uptime KPI time series (demo)



---



\### generated/partner/



\*\*Produced by\*\*

\- `tabs/partner\_data\_tab.py`



\*\*Files\*\*

\- `partner\_cable\_metrics.<n>.csv`



\*\*Purpose\*\*

\- Simulated partner-provided datasets

\- Used for correlation and integration demos



---



\### generated/metadata/



\*\*Produced by\*\*

\- `app.py`



\*\*Purpose\*\*

\- Tab, service, and workpackage metadata

\- Used for UI associations and documentation



---



\## IMPORTANT RULES


\- No tab or logic module should create directories manually

\- No hard-coded `"data/..."` paths outside `utils/paths.py`

\- Deleting `generated/` must never break the application
